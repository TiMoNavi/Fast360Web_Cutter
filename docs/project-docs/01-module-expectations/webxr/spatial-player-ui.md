# 空间播放器 UI

## 定位

播放器 UI 是悬浮在空中、面向用户的空间 UI。它负责“看视频”和“换视频”，不承载剪辑器的大块操作。

第一版功能优先，不要求视觉精致，但结构要为后续 visionOS 风格打基础。

## UI 内容

播放器 UI 提供：

```text
视频列表。
当前视频标题。
播放 / 暂停。
上一条 / 下一条。
进度条。
当前时间 / 总时长。
seek。
倍速显示。
全隐藏 / 显示。
进入 / 退出 VR 状态提示。
加载和错误状态。
```

不放在播放器 UI 的内容：

```text
Cut。
锁定 / 解锁。
放弃 / 恢复。
保存路径。
导出参数。
特效事件编辑。
复杂 FOV 参数面板。
```

这些属于剪辑工作台。

## 空间位置

默认位置：

```text
用户正前方。
略低于视线中心或位于视频下方。
面板始终面向用户。
不遮挡中央取景区域。
```

播放器 UI 可以整体隐藏。隐藏后只保留极简唤回按钮或 controller 快捷键，避免观看和裁剪时被面板挡住。

## 视频列表

视频列表用于选择素材，不用于上传。列表项至少显示：

```text
标题 / 文件名。
时长。
当前 session 状态。
是否已有最近导出。
```

选择视频后，播放器切换到对应 source，并保持 WebXR 端只通过后端 API 获取业务数据。

## 进度条

进度条需要支持：

```text
显示当前播放进度。
点击或拖动 seek。
显示缓冲状态。
在隐藏播放器 UI 时停止占用中心视野。
```

seek 后采样层应知道用户发生了时间跳转。是否覆盖旧路径由 `ViewPathPatch.replaceRange` 策略处理，不由进度条直接决定。

## 全隐藏

全隐藏是播放器 UI 的核心能力：

```text
隐藏视频列表。
隐藏播放按钮。
隐藏进度条。
隐藏状态面板。
保留最小唤回方式。
不隐藏裁剪遮罩和必要 reticle。
不关闭剪辑工作台的内部状态。
```

全隐藏只影响 UI 可见性，不影响播放状态和路径采样。
