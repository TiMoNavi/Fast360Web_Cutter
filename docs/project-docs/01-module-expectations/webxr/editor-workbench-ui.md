# 剪辑器工作台 UI

## 定位

剪辑器组件全部放在“工作台”上。工作台相对用户的位置像桌面：位于前下方、略微倾斜，用户可以低头看到并用 controller 点击。

工作台负责剪辑动作和参数，不负责视频列表和基础播放控制。

## 布局

第一版工作台结构：

```text
前下方主桌面：
大块模块入口按钮。

弹出模块：
点击大按钮后，从工作台上方弹起对应模块 UI。

中心视野：
保持 360 视频、取景框、reticle 和遮罩，不放大块编辑面板。
```

工作台不应该做成传统桌面网页控制台。它是空间里的操作台，按钮数量要少，按钮尺寸要大，交互要能被 Quest controller 稳定点中。

## 大块入口按钮

第一版大按钮：

```text
取景
Cut
FOV
锁定
放弃 / 恢复
采样
保存
特效
导出
Session
```

大按钮只负责打开模块或执行明确单步动作。复杂参数在弹出模块里处理。

## 弹出模块

点击工作台按钮后，弹出对应模块：

```text
FOV 模块：
显示当前 FOV，提供增减或滑动控制。

采样模块：
显示采样中 / 暂停、最近保存状态、待上传点数。

特效模块：
创建时间范围事件，填写名称和少量参数。

导出模块：
显示导出参数和导出状态。

Session 模块：
显示当前 videoId、sessionId、take 历史和恢复入口。
```

同一时间默认只打开一个弹出模块，避免空间里堆叠太多面板。

## 高频动作

高频剪辑动作不完全依赖点击面板。保留 controller 快捷动作：

```text
按住 trigger：
让取景框跟随 controller ray 或 head-gaze。

松开 trigger：
锁定当前取景状态。

右手 thumbstick 上 / 下：
调整 FOV。

右手 thumbstick 左 / 右：
切换播放倍速。

按住 A 或 thumbstick click：
打开剪辑环形菜单。
```

工作台按钮用于清楚展示状态和提供低频入口，controller 快捷动作用于连续裁剪。

## 与播放器 UI 的区别

```text
播放器 UI：
视频列表、播放、暂停、进度、隐藏。

剪辑工作台：
取景、FOV、Cut、采样、保存、特效、导出。
```

两个 UI 可以同时存在，但播放器 UI 必须可隐藏，工作台必须保持前下方桌面感。
