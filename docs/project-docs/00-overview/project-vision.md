# 项目总述

## 项目名称

```text
The Invisible Director
```

它的目标不是做一个普通 360 视频播放器，也不是做一个传统桌面剪辑软件，而是让用户在沉浸式 WebXR 环境里像“隐形导演”一样观看 360 视频，并把自己的观看意图变成一条可渲染的平面视频取景路径。

## 产品概念

360 视频的问题是：素材包含完整空间，但最终观众通常观看的是普通平面视频。传统做法需要在桌面时间线上手动打关键帧、调 yaw / pitch / FOV。这个项目希望把这个动作变得更自然：

```text
用户戴上 Quest 或进入 WebXR 环境。
用户直接观看 360 视频。
头显视线、手柄指向、取景框和快捷动作共同决定“最终视频应该看哪里”。
系统把这些意图压缩成低频取景路径。
后端根据路径重新投影 360 视频，导出普通 16:9 MP4。
```

这个产品里的“导演”不是实时录屏者，而是路径的创建者。用户在 VR 中做的是取景决策，后端做的是正式裁剪和导出。

## 目标用户流程

```text
1. 用户在手机或普通浏览器打开 Web 页面。
2. 登录或注册。
3. 上传 360 MP4。
4. 在“我的视频”中创建一个 WebXR 裁剪 session。
5. 用 Quest 打开 WebXR 入口。
6. 在 360 球幕中播放视频。
7. 用头显、手柄和取景框决定输出画面中心。
8. 系统低频保存 ViewPathPatch。
9. 后端按路径渲染导出 MP4。
10. 用户回到普通 Web 页面查看进度并下载结果。
```

## 三个模块

项目由三个主要模块组成：

```text
WebXR：
唯一裁剪入口。负责沉浸式播放、取景框、路径采样和路径上传。

Web：
素材入口和结果出口。负责登录、上传、视频列表、WebXR 入口、状态和下载。

Backend：
唯一正式裁剪入口。负责认证、视频存储、路径时间线、渲染任务、导出文件和下载。
```

## 核心原则

```text
Web 端不做裁剪。
WebXR 端不生成最终 MP4。
Backend 不依赖前端组件内部状态。
正式导出只读取稳定协议，而不是读取某个页面的临时状态。
```

最重要的协议是：

```text
ClipEditConfig
ViewPathPatch
ViewPathPoint
PlaybackClientState
```

其中 `ClipEditConfig` 和 `ViewPathPatch / ViewPathPoint` 是正式裁剪输入。`PlaybackClientState` 只用于播放体验、调试和 UI 恢复，不参与正式导出。

## 项目现在处于什么阶段

当前项目已经具备一个开发 MVP 闭环：

```text
登录
上传视频
创建 session
发送测试 ViewPathPatch
后端保存路径
同步 render-test 导出短 MP4
下载结果
```

但它还没有完成真正的 WebXR 裁剪体验。真实头显姿态采样、controller 操作、取景框遮罩、5Hz sampler、生产级 60 秒分片队列仍是后续重点。

```text

```
