# 5 分钟讲解结构 v2 - 酷炫优先版

## 核心叙事逻辑

```text
不是"发现问题 → 解决问题"
而是"这很酷 → 这有用 → 这可行"
```

## 总节奏

```text
0:00-0:30  我们做了什么：定义 The Invisible Director
0:30-1:45  酷炫时刻：戴上头显，看一遍就剪出来
1:45-2:30  为什么有价值：从观看到分享的最短路径
2:30-3:15  技术实现：ViewPath 注意力路径
3:15-4:15  当前完成度：PC 端、试用入口、真实闭环
4:15-5:00  定位收束：学生探索 + 影石友好
```

---

## 0:00-0:30 开场定义

**目标**：用产品名制造爆点，直接告诉评委我们做了什么。

**要讲**：

```text
我们的项目名是"一眼成片"。

就像它的名字一样，这是一个极其轻量的 Web 360 度视频剪辑器。
我们戴上头显，欣赏一遍视频，就能快速成片。

简单说：看一遍，就能发。
```

**PPT 对应**：第 1 页
- 标题：一眼成片
- 副标题：看一遍，就能发
- 英文：OneGlance / At First Sight
- 视觉：VR 头显 + 360 视频 → 16:9 成片

## 0:30-1:45 酷炫演示

**目标**：让项目先酷起来。这是整个演讲的高光时刻。

**要讲**：

```text
现在我戴上头显，进入一段 360 素材。

[播放 VR 实机录屏]

你看到的这个 16:9 框，就是最终观众会看到的画面。

我转头看向这里 → 系统记录：观众应该看这里
我按一下 Lock → 系统锁定这个主体，自动跟随
这段没意思，我按 Skip → 这段不会进成片
这里需要节奏，我按 Cut → 系统在这里切一刀
我想强调这个细节 → 我调 FOV 推近

看完了。我按导出。

[切到 PC 端或手机端]

系统后端按我刚才的注意力路径重新渲染。
几秒钟后，一个普通 16:9 MP4 就生成了。
我可以直接下载，发到任何平台。

整个过程：戴上头显，看一遍，按几个按钮，拿到成片。
```

**PPT 对应**：第 2-3 页
- 第 2 页：VR 实机截图 + 操作标注（Look / Lock / Skip / Cut / FOV）
- 第 3 页：导出流程图（VR 观看 → ViewPath → 后端渲染 → MP4 下载）

## 1:45-2:30 价值说明

**目标**：从"酷"过渡到"有用"。

**要讲**：

```text
为什么要做这个？

360 相机能拍下完整现场，但最终观众看的还是普通视频。
用户必须决定：观众看哪里、哪里跳过、哪里切、哪里推近。

传统流程是：
拍完 → 导入电脑 → 打开剪辑软件 → 拖时间轴 → 调关键帧 → 导出

我们的流程是：
拍完 → 戴上头显 → 看一遍 → 拿到成片

这是从观看到分享的最短路径。
很多素材不是拍得不好，而是卡在"我懒得剪"。
现在，看一遍就能发。
```

**PPT 对应**：第 4 页
- 对比图：传统流程 vs 我们的流程
- 核心短句："从观看到分享的最短路径"

## 2:30-3:15 技术实现

**目标**：把酷炫体验落到可信工程。

**要讲**：

```text
这里不是录屏。

WebXR 记录的是 ViewPath，也就是注意力路径。
它包含：
- 每个时间点你看向哪里（yaw / pitch）
- 你按了什么按钮（Lock / Skip / Cut）
- 你调了什么参数（FOV）

后端拿到这条路径，按路径重新投影 360 素材，
导出成普通 16:9 MP4。

这意味着：
- 不损失画质（不是录屏）
- 可以重新渲染（路径可编辑）
- 可以接入 AI（路径是结构化数据）
```

**PPT 对应**：第 5 页
- ViewPath 数据结构示意图
- 技术关键词：WebXR / ViewPath / 重投影 / AI-ready

## 4:15-5:00 定位收束

**目标**：回到学生作品气质，同时对影石友好。

**要讲**：

```text
这不是一个成熟商业产品。
它是一个黑客松里对空间智能工作流的探索。

影石这样的相机已经解决了拍到全部。
我探索的是拍完以后，如何更快确定观众注意力。

成熟工具适合完整剪辑和精修。
我的原型放在更轻的一层：第一次观看素材时就生成取景意图。

我想证明的是：
360 素材可以从观看直接走向分享。
看一遍，就能发。

谢谢。
```

**PPT 对应**：第 8 页
- 核心短句："看一遍，就能发"
- 定位说明：学生探索 / 影石友好 / 空间智能工作流
- 致谢

## 与 v1 的核心差异

| 维度 | v1（问题优先） | v2（酷炫优先） |
|------|---------------|---------------|
| 开场 | 360 素材难发出去（痛点） | 我们做了什么（定义） |
| 第一高潮 | 定位说明（防御） | VR 演示（酷炫） |
| 价值说明 | 前置（0:40） | 后置（1:45） |
| 技术细节 | 中后段 | 中后段（不变） |
| 情绪曲线 | 问题 → 方案 → 演示 | 演示 → 价值 → 技术 |

v2 的核心逻辑：**先让评委觉得酷，再让评委觉得有用，最后让评委觉得可信。**
