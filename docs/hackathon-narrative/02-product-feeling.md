# 产品感觉

## 它应该是什么感觉

这个项目应该先让人觉得：

```text
原来 360 视频就应该这样剪。
```

不是一个复杂专业软件，不是堆满按钮的 VR 工作站，也不是一个只能演示技术的播放器。它应该像一个轻量、直觉、带一点未来感的空间取景工具。

用户进入后，第一感觉应该是：

```text
我站在视频里。
我手上有一个取景器。
我可以边看边决定最终画面。
```

## 酷在哪里

酷点不在于 UI 有多花，而在于交互逻辑变了。

传统剪辑是：

```text
先看素材。
回到时间线。
找片段。
打关键帧。
调参数。
反复预览。
再改。
```

这个项目想变成：

```text
播放开始。
我看。
我按。
我锁。
我丢。
我推近。
播放结束。
成片路径完成。
```

酷点是“剪辑动作和观看动作合并了”。用户不再先看再剪，而是在观看过程中自然完成剪辑。

## 轻在哪里

轻量不是功能少，而是决策负担少。

第一版体验应该尽量避免：

```text
密集时间线。
复杂轨道。
大量参数面板。
需要学习的专业术语。
每一步都弹窗确认。
```

更适合的感觉是：

```text
中心视野只保留视频和取景框。
常用操作在手柄快捷按钮上。
状态提示短而明确。
需要时才出现工作台。
不需要时 UI 可以隐藏。
```

## 新奇在哪里

新奇感来自一个简单反转：

```text
不是把 360 视频压回平面屏幕上剪。
而是让剪辑者进入 360 视频里面剪。
```

这个反转很容易让人理解，也适合黑客松现场演示。评委不需要懂完整架构，只要看到一个人戴着头显在球幕里边看边取景，就能理解它和传统剪辑方式不一样。

## 低负担在哪里

用户不应该感觉自己在“操作软件”，而应该感觉自己在“看视频，只是顺手做了几个决定”。

所以交互应该优先支持高频、低思考成本的动作：

```text
保留这一段。
丢掉这一段。
在这里切一下。
现在看这里。
这个角度锁住。
推近一点。
拉远一点。
保存当前路径。
```

这些动作都应该是一按就懂的动作，而不是需要进入二级菜单的参数配置。

## 和纯 AI 剪辑的区别

这个项目不需要把自己讲成“AI 自动替你剪完”。它更有意思的点是：

```text
人类在空间里做判断。
系统把判断变成可渲染路径。
```

AI 可以未来参与辅助，比如推荐主体、自动平滑、自动补帧，但核心体验仍然是 human-in-the-loop：

```text
最终画面不是机器猜的。
是你看过、选过、确认过的。
```

## 体验目标

第一版最重要的体验目标：

```text
用户能在 30 秒内理解这个产品。
用户能在 1 分钟内学会核心操作。
用户能在看完一遍素材后得到一条可导出的剪辑路径。
旁观者能一眼看出这不是普通播放器。
```
