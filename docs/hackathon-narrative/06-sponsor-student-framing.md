# 赞助商与学生作品表达

## 面对影石赞助商

核心态度：

```text
尊重成熟产品，不正面挑战。
强调生态补充，不说替代。
把项目放在“快速生成取景意图”的前置层。
```

推荐说法：

```text
影石这样的相机已经让用户可以拍到完整现场。
但拍完以后，用户仍然需要决定观众到底看哪里。
我的原型探索的是这个轻量环节：第一次观看素材时，顺手生成注意力路径。
成熟工具适合精修，我这个原型适合快速进入发布前的取景草稿。
```

不要说：

```text
我做了一个比现有工具更快的 360 剪辑器。
我想替代影石 Studio。
影石缺少这个功能，所以我来补。
```

## 面对学生作品现实

核心态度：

```text
不装成熟商业产品。
不回避当前 MVP 边界。
强调黑客松最重要的是方向、演示和闭环。
```

推荐说法：

```text
这是一个学生黑客松原型。
我不想把它包装成完整商业工具。
它现在最重要的价值是证明一种空间智能工作流：
用户在 360 素材里观看，系统记录取景意图，后端导出普通视频。
```

可以强调：

```text
酷炫。
直觉。
WebXR 原生。
有真实后端闭环。
未来适合 AI 先生成路径、人再确认。
```

不要强调：

```text
商业规模。
商业模式。
完整产业闭环。
替代专业软件。
已经生产可用。
```

## 酷炫但不虚夸

酷炫来自这个画面：

```text
用户戴上头显，站进 360 素材里。
他看哪里，最终视频就看哪里。
他按几个按钮，系统就形成注意力路径。
```

可信来自这个边界：

```text
WebXR 不录屏。
WebXR 记录 ViewPath。
后端正式重投影导出 MP4。
当前 MVP 已跑通上传、session、路径提交、smoke render、下载。
```

## AI + 空间智能表达

推荐说法：

```text
AI 不一定要直接替用户剪完。
AI 可以先根据人物、运动、声音或画面变化生成初版注意力路径。
人再在 WebXR 里像导演一样确认：看哪里、切哪里、跳过哪里。
```

这样讲，AI 是工作流的一部分，不是空喊“自动剪辑”。

## 现场语气

整体语气应该是：

```text
自信，但不硬碰成熟产品。
酷炫，但不夸大完成度。
学生气，但不是玩具。
尊重赞助商，同时展示自己的新角度。
```
