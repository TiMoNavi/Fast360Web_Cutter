# Q&A 与不要说的话

## 评委问：这和影石自己的工具有什么区别？

回答：

```text
我不把它定义成影石 Studio 的替代品。
成熟工具适合完整后期和精修。
我探索的是更轻的前置层：用户第一次观看 360 素材时，快速生成取景意图和注意力路径。
这条路径后续可以交给后端导出，也可以进入更成熟的工具继续精修。
```

## 评委问：为什么一定要 WebXR？

回答：

```text
360 素材本身是空间内容，取景决策本质上也是空间判断。
在 2D 时间线上调 yaw、pitch、FOV，是用参数描述空间判断。
WebXR 让用户直接站回素材里，用看向和指向表达“观众应该看哪里”。
```

## 评委问：AI 在哪里？

回答：

```text
我没有把 AI 包装成黑箱自动剪辑。
更适合的方式是 AI 先生成初版注意力路径，比如根据人物、运动、声音高潮找到候选重点。
人再在 WebXR 里确认和修正。
所以 AI 和人的空间判断共同生成 ViewPath。
```

## 评委问：现在完成到什么程度？

回答：

```text
当前 MVP 已经跑通上传、session、路径提交、后端 smoke render 和下载。
还在补齐的是真实头显姿态采样、controller 快捷按钮、取景框遮罩、5Hz sampler 和生产级分片渲染队列。
今天我展示的是核心闭环和目标交互，而不是宣称已经完整生产可用。
```

## 评委问：为什么不直接录屏？

回答：

```text
录屏是一次性的画面捕获，后续很难覆盖、重渲染或交给 AI 修改。
我记录的是 ViewPath：时间点、视角、FOV、跳过、切段和锁定。
后端按路径重新投影，所以它更像剪辑管线，而不是屏幕捕获。
```

## 评委问：二维码能体验什么？

回答：

```text
二维码指向现场可访问的真实入口或备用演示页。
它用于补充试用和可信度，主展示仍然以提前录好的实机视频为准。
如果现场网络不稳定，我会用视频和 PC 截图完整展示闭环。
```

## 不要说的话

不要说：

```text
这是影石 Studio 的替代品。
这是完整商业级 360 剪辑器。
AI 会自动帮用户剪完一切。
用户完全不需要判断。
WebXR 直接输出最终 MP4。
一键分享到所有平台已经完成。
生产级长视频渲染已经完成。
```

改成说：

```text
这是最轻空间导演层。
它生成取景意图和注意力路径。
后端负责正式重投影导出。
当前 MVP 已经跑通核心闭环。
真实头显体验和分享出口还在继续补齐。
```

## 允许保留的辅助句

```text
看一遍，就剪完。
```

这句话可以作为爽点，但不要作为主定位。主定位应该是：

```text
看一遍，就能发。
从观看 360，到分享成片。
```
