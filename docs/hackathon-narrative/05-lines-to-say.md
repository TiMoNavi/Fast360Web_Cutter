# 可以直接说的话

这些句子不是最终稿，可以在路演、讲解、README 或视频脚本里挑着用。

## 核心句

```text
看一遍，就剪完。
```

```text
360 视频本来就是空间素材，所以我想用空间方式剪它。
```

```text
传统方法是在 2D 屏幕上猜视角；这个项目让你直接进入视频里决定视角。
```

```text
它不是一个普通 360 播放器，而是一个 VR 里的轻量取景剪辑工具。
```

```text
你看到哪里、留下哪里、丢掉哪里，都会变成一条可渲染的取景路径。
```

## 痛点句

```text
360 相机把整个现场都拍下来了，但最后观众只能看一个画面。
```

```text
后期最难的不是播放 360 视频，而是决定最终视频应该看哪里。
```

```text
在平面时间线上调 yaw、pitch、FOV，本质上是在用参数描述一个空间判断。
```

```text
很多 360 素材不是拍得不好，而是后期取景太麻烦。
```

## 产品句

```text
用户不需要先学一个复杂剪辑软件，只要进入 WebXR 链接，开始看视频。
```

```text
中心视野是视频，手上是快捷按钮，眼前是最终取景框。
```

```text
保留、丢弃、切段、锁定、推近，都应该是播放过程里的自然动作。
```

```text
我想让剪辑动作变得像观看一样自然。
```

```text
播放不是预览，播放本身就是剪辑过程。
```

## 技术边界句

```text
WebXR 端不生成最终 MP4，它只记录剪辑意图。
```

```text
后端正式读取的是 ViewPath，而不是前端页面的临时状态。
```

```text
这不是录屏，而是把空间观看过程变成可回放、可覆盖、可重新渲染的路径。
```

```text
路径低频保存，渲染高质量交给后端。
```

## 不要太商业时的说法

```text
我先不把它讲成一个商业插件，我更想证明一种新的剪辑方式。
```

```text
这个项目最重要的不是适配某个平台，而是验证 VR 和 360 视频之间天然的剪辑关系。
```

```text
如果 360 视频是空间记录，那么 VR 剪辑就是很自然的下一步。
```

## 收尾句

```text
我的目标不是做一个更重的剪辑软件，而是让 360 视频第一次可以被很轻地、很直觉地剪完。
```

```text
我希望评委看到这个 Demo 时，第一反应不是“功能很多”，而是“对，360 视频就应该这样剪”。
```

```text
看一遍，就剪完。这就是这个项目最想证明的事情。
```
