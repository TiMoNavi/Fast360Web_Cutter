# VR 原生 360 剪辑叙事

这组文档不是正式产品规格，也不是商业计划书。它是一组黑客松前的叙事草稿，用来帮助项目讲清楚一个更直接、更酷的观点：

```text
看一遍，就剪完。
```

360 视频本来就是空间内容。它记录的不是一个固定画面，而是一个完整现场。传统后期却经常要求创作者回到 2D 屏幕上，用时间线、关键帧、yaw、pitch、FOV 去“猜”自己当时应该看哪里。

这个项目想反过来：

```text
进入 360 视频空间。
像观众一样看一遍。
像导演一样决定画面。
系统把观看、取景、保留、丢弃、锁定这些动作变成剪辑路径。
播放结束，剪辑意图也已经完成。
```

## 核心叙事

项目的重点不是做一个更复杂的剪辑软件，而是提出一种更自然的 360 视频剪辑方法：

```text
用 VR 剪 360 视频。
```

原因很简单：

```text
360 视频是空间素材。
VR 是空间观看方式。
剪辑决策本质上是“我想让观众看哪里”。
所以最自然的剪辑入口，应该发生在空间里。
```

## 当前事实和预期体验

这组文档允许讨论“预期体验”，但表达时要区分当前 MVP 和最终形态。

当前项目已经验证的是：

```text
上传 360 视频。
创建 WebXR cut session。
提交 ViewPathPatch。
后端保存路径点。
后端按路径 smoke render 导出 MP4。
下载结果。
```

仍在建设的是：

```text
真实头显姿态采样。
controller 快捷操作。
取景框和遮罩。
边播边记录的 5Hz sampler。
生产级分片渲染队列。
```

所以路演叙事可以讲“我们要实现的 VR 原生剪辑方式”，但不要把未完成能力说成已经完整生产可用。
